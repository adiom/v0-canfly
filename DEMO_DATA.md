## Структура данных

### Таблица `books`
```sql
SELECT 
  id, title, slug, type, price, is_featured, display_order
FROM books
ORDER BY display_order;
```

Результат:
- 5 книг/комиксов
- Все помечены как featured (видны в слайдере)
- Отсортированы по display_order от 1 до 5

### Таблица `characters`
```sql
SELECT 
  id, name, slug, bio
FROM characters
ORDER BY created_at DESC;
```

Результат:
- 5 персонажей
- Каждый с именем, slug, биографией и аватаром

### Таблица `character_relationships`
```sql
SELECT 
  c1.name as from_character,
  c2.name as to_character,
  relationship_type,
  description
FROM character_relationships cr
JOIN characters c1 ON cr.character_id = c1.id
JOIN characters c2 ON cr.related_character_id = c2.id;
```

Результат:
- 10+ связей между персонажами
- Типы: Союзник, Наставник, Противник, Боевой товарищ и т.д.

## Тестирование функциональности

### Когда вы открываете сайт:

1. **Главная страница** (`/`)
   - Увидите 5 книг в слайдере (все featured)
   - Сможете переключаться между ними стрелками
   - Под слайдером 5 кнопок с названиями книг

2. **Страница персонажей** (`/characters`)
   - В центре граф с 5 узлами (персонажи)
   - Линии между ними показывают связи
   - Ниже граф 5 карточек персонажей
   - Кликните на любого → откроется полный профиль

3. **Магазин** (`/shop`)
   - Сетка 5 товаров
   - Нажмите "В корзину" → товар добавится (localStorage)
   - Цена отобразится в ₽

4. **Корзина** (`/cart`)
   - Все добавленные товары
   - Форма с полями: имя, email, телефон, адрес
   - При отправке заказ сохранится в Supabase

5. **Читалка** (`/books/cipher-chronicles-genesis`)
   - Тестовые страницы будут пусты (нет preview_pages)
   - Можно добавить их вручную через админку

6. **Админ-панель** (`/admin`)
   - Вход: admin@canfly.local / admin123
   - Три раздела с данными из БД
   - Просмотр всех заказов

## Добавление дополнительных данных

### Добавить книгу

```sql
INSERT INTO books (title, slug, type, description, price, is_featured, display_order)
VALUES (
  'Новая книга',
  'novaya-kniga',
  'book',
  'Описание новой книги',
  89999,
  false,
  6
);
```

### Добавить персонажа

```sql
INSERT INTO characters (name, slug, bio, full_description)
VALUES (
  'Новый герой',
  'novyy-geroy',
  'Краткое описание',
  'Полное описание персонажа'
);
```

### Связать персонажей

```sql
INSERT INTO character_relationships (character_id, related_character_id, relationship_type, description)
VALUES (
  (SELECT id FROM characters WHERE slug = 'cipher'),
  (SELECT id FROM characters WHERE slug = 'novyy-geroy'),
  'Товарищ',
  'Они встречаются в ключевой момент'
);
```

## Где находятся данные?

### В Supabase
- **URL**: https://your-project.supabase.co
- **Таблицы**: books, characters, character_relationships, orders, admins
- **RLS**: Включен для безопасности

### На фронтенде
- **Корзина**: localStorage (browser storage)
- **Токен админа**: localStorage['admin-token']

### В приложении
- **API**: `/api/books`, `/api/characters`, `/api/orders`, `/api/admin/*`
- **Routes**: Все страницы используют server components для кеширования

## Примечания

- Все цены в **копейках** (599.99 ₽ = 59999 копеек в БД)
