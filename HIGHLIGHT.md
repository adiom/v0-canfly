# HIGHLIGHT.md - Функциональность выделений и правок

Этот документ описывает реализованную функциональность выделений (highlights) и правок в системе Canfly.

## ✨ Ключевые возможности

### 1. Поддержка ролей
Система поддерживает следующие роли пользователей, которые влияют на видимость и возможности работы с выделениями:
- **`reader`**: Может создавать публичные цитаты.
- **`editor`**: Может создавать редакторские комментарии (`editorial_comment`), видимые только команде (авторам, администраторам, другим редакторам).
- **`author`**: Видит все внутренние комментарии (от редакторов и свои) и может их разрешать/игнорировать.
- **`admin`**: Имеет полный доступ ко всем типам выделений и может управлять их статусом и видимостью.

### 2. Типы выделений
- **`quote` (Цитата)**: Выделяется читателем и видна публично. Может быть опубликована в профиле пользователя и использоваться для шаринга в соцсетях.
- **`editorial_comment` (Правка редактора)**: Выделяется редактором/администратором. Видна только внутренней команде (авторам, редакторам, администраторам). Содержит комментарий к выделенному фрагменту.
- **`author_note` (Заметка автора)**: Выделяется автором/администратором. Видна только внутренней команде (авторам, редакторам, администраторам).

### 3. База данных
Создана новая таблица `highlights` для хранения всех выделений. Основные поля:
- `id`: UUID (уникальный идентификатор выделения/правки).
- `book_id`: Ссылка на книгу.
- `user_id`: Пользователь, который сделал выделение.
- `chapter_index`: Номер главы, в которой сделано выделение.
- `text_content`: Текст, который был выделен.
- `comment`: Дополнительный комментарий (для редакторских правок).
- `type`: Тип выделения (`quote`, `editorial_comment`, `author_note`).
- `visibility`: Уровень видимости (`public`, `internal`, `private`).
- `status`: Статус обработки правки (`pending`, `resolved`, `ignored`). Актуально для `editorial_comment` и `author_note`.
- `range_data`: JSONB поле для хранения точных данных о выделении (начало/конец, смещения).

### 4. API Эндпоинты
- `GET /api/highlights?bookId=[id]`: Получение всех выделений для конкретной книги, с учетом прав доступа текущего пользователя.
- `POST /api/highlights`: Создание нового выделения. Автоматически определяет `type` и `visibility` на основе роли пользователя.
- `GET /api/highlights/[id]`: Получение конкретного выделения по его UUID, с проверкой прав доступа.
- `PATCH /api/highlights/[id]`: Обновление статуса (только для авторов/админов/редакторов) или комментария/видимости (для владельца/админа).
- `DELETE /api/highlights/[id]`: Удаление выделения (только для владельца/админа).

### 5. Frontend (`components/book-reader.tsx`)
- **Инициализация:** Компонент `BookReader` теперь принимает `initialHighlights` с сервера для быстрой загрузки.
- **Определение ролей:** Получает текущего пользователя и его роли для динамического отображения интерфейса.
- **Выделение текста:** При выделении текста в режиме чтения/редактирования появляется плавающее меню:
    - **`reader`**: может создать публичную цитату.
    - **`editor`/`admin`**: может добавить редакторский комментарий к выделенному тексту (с полем для ввода комментария).
- **Список правок/цитат:** Внизу каждой главы отображается список существующих выделений.
    - Для внутренних правок (от редакторов/авторов) доступны кнопки `Решено` / `Игнорировать` (для авторов/админов).

### 6. Профиль пользователя (`app/profile/page.tsx`)
- На странице профиля пользователя (`/profile`) добавлен новый раздел `Мои цитаты`, где отображаются все `public` выделения типа `quote`, сделанные данным пользователем. Каждая цитата имеет ссылку `Поделиться`.

### 7. Экспорт для аудиокниг (`app/api/books/[slug]/export/route.ts`)
- Добавлен эндпоинт `GET /api/books/[slug]/export`, который позволяет получить чистый текстовый контент книги (или JSON) для использования во внешних сервисах, таких как Eleven Labs.
